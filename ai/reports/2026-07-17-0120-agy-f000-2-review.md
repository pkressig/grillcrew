# AGY Review - F000.2 AI Project Memory and Multi-Agent Workflow

- Agent: AGY / Antigravity
- Timestamp: 2026-07-17T01:20:00+02:00
- Branch: feature/f000-ai-project-memory
- Assignment: Product, workflow and usability review of F000.2

## Verdict

AGY marked the initial F000.2 version as not ready to commit.

## Findings

Critical / blocking:

1. Context pack gap

   `scripts/build-ai-context.mjs` embedded the AI core files but only listed
   authoritative documents such as `docs/PRD.md` and `docs/DECISIONS.md` as
   bullet paths. If GitHub connector access is unavailable, ChatGPT or another
   external agent would not actually receive those documents.

2. Hardcoded workstation paths and OS assumptions

   `README.md` documented startup commands using the workstation-specific path
   `C:\Users\pkres\Documents\GitHub\grillcrew` and Windows-specific command
   examples without enough cross-platform guidance.

3. German legacy prompt template

   `prompts/01-bootstrap-claude-code.md` was still written in German and no
   longer matched the current English, feature-scoped workflow.

## Product Owner Decisions

Approved fixes:

- Embed authoritative documentation and current feature plans in the generated
  context pack.
- Replace hardcoded workstation paths with `<your-repository-root>` and
  document when to use `npm.cmd` versus `npm`.
- Mark the old bootstrap prompt as deprecated historical context in English
  instead of treating it as an active prompt.

## Fixes Applied After Review

- `scripts/build-ai-context.mjs` now embeds authoritative documents and current
  feature plans in `ai/generated/CONTEXT_PACK.md`.
- `README.md` no longer hardcodes the local workstation path and documents the
  installed `agy` command.
- `prompts/01-bootstrap-claude-code.md` is now an English deprecated legacy
  prompt that points agents to the current workflow files.

## Result

After the approved fixes, F000.2 should be reviewed through the normal
validation flow before commit.
